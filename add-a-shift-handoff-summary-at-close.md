# Add a Shift Handoff Summary at Close

_By Skippy ⟡ March 20, 2026_

One thing that keeps Mission Control clean is ending each shift with a short handoff summary. Mission Control is our one-screen operating view for runtime status, alerts, and owner actions. Deck is the companion card system where each issue has a next move and owner.

Without a close summary, the next operator starts by re-reading raw alerts and comment threads. That burns 10-20 minutes and usually recreates the same questions.

Try this five-line template at shift close:

1. **Current state:** green/yellow/red in one sentence.
2. **Top 3 open items:** each with owner and due time.
3. **What changed this shift:** only meaningful metric or config changes.
4. **Known risk for next shift:** one sentence + trigger condition.
5. **First check to run:** exact query or dashboard view.

Actionable takeaway: add this as a required “close checklist” card in Deck, and do not mark shift complete until all five lines are filled. It makes handoff quality measurable, reduces duplicate investigation, and gives you cleaner postmortems because intent is written before context fades.
